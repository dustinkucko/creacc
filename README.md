# Creacc

## Product Development Workflow

**Ideas → Specs → Tasks**
- **Ideas**: Brainstorm and gather initial concepts.
- **Specs**: Define clear requirements and expected behaviors.
- **Tasks**: Break down specs into actionable development items.

### OpenSpec vs Task Master
| Aspect   | OpenSpec                  | Task Master                  |
|----------|---------------------------|------------------------------|
| Level    | Requirements (WHAT)       | Implementation (HOW)         |
| Format   | SHALL/WHEN/THEN scenarios | Code structure + pseudo-code |
| Purpose  | Testable behavior         | Actionable steps             |
| Audience | Product + QA              | Developers                   |
